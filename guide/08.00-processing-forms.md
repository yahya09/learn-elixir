## The Form Lifecycle

Understanding how forms flow through Phoenix:

```
User submits form
       ↓
POST /snippets (with CSRF token)
       ↓
Router → SnippetController.create
       ↓
Build changeset from params
       ↓
Validate changeset
       ↓
  Valid? ─────────────────┐
    │                     │
   Yes                   No
    │                     │
    ↓                     ↓
Insert to DB        Re-render form
    │               with errors
    ↓                     │
Redirect to             Return
show page           changeset errors
```

## Form Components Preview

Phoenix 1.7+ uses function components for forms:

```heex
<.form for={@changeset} action={~p"/snippets"}>
  <.input field={@form[:title]} label="Title" />
  <.input field={@form[:content]} type="textarea" label="Content" />
  <.input field={@form[:language]} type="select" label="Language"
    options={["Elixir", "JavaScript", "Python", "Ruby"]} />

  <.button>Create Snippet</.button>
</.form>
```

This generates:
- Proper HTML structure
- CSRF token automatically included
- Error messages displayed
- Accessible labels and inputs
